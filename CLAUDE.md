# AniTracker — Claude Code Instructions

## MANDATORY — Service Worker Cache

**The `pre-commit` hook auto-bumps `CACHE_NAME` in `sw.js` on every commit.**

- Hook: `.githooks/pre-commit`
- Config: `git config core.hooksPath .githooks` (repo-local)
- Increments `anitracker-vN` → `anitracker-v(N+1)` and re-stages `sw.js`

**On a new machine/session**, run once:
```
git config core.hooksPath .githooks
```

Never edit `CACHE_NAME` manually — the hook handles it.

## After every push

Always display the PR creation link:

```
**[→ Create PR on GitHub](https://github.com/wpef/anitracker/pull/new/BRANCH_NAME)**
```

Replace `BRANCH_NAME` with the current branch.

- `gh` is not available — cannot create PRs automatically
- Ignore the "view PR" button generated by Claude Code (points to old closed PR)

## Project overview

Dog activity tracker PWA. Firebase Realtime DB or in-memory demo mode.
Vanilla JS ES modules, no framework, no build step.

### File structure

```
index.html              Main SPA (all pages as hidden <section>s)
quick.html              Standalone quick-entry page (Android shortcut)
css/style.css           Single stylesheet, CSS variables in :root
sw.js                   Service Worker (offline cache)
js/                     ES modules (see js/CLAUDE.md for full architecture)
AGENT-NEW-ACTION.md     Guided prompt for the "add new action type" agent
```

### Adding a new entry type

**Single file to edit: `js/utils.js` — add an entry to `TYPE_DEF`.**

All UI modules (forms, history, stats, quick page, edit page) are fully
data-driven from `TYPE_DEF`. No other file needs to change.

See `AGENT-NEW-ACTION.md` for a step-by-step agent prompt, or
`js/CLAUDE.md` for the `TYPE_DEF` schema reference.

## Conventions

- **UI strings**: French. **Code/comments/docs**: English
- **No build step**: import paths must include `.js` extension
- **CSS**: single file `css/style.css`, no preprocessor, CSS custom properties in `:root`
- **External deps**: Chart.js (CDN), Firebase SDK (CDN) — nothing else
- **Module naming**: `ui-*.js` = DOM-touching, others = pure logic or shared utilities
- **Entry data schema**: all types use `BaseEntry` (see `js/utils.js` typedef).
  Generic fields `text_val` (string) and `num_val` (0–100) replace old named fields.
